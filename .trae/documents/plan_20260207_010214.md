## 问题确认
用户反馈详情页右侧出现蓝色背景，且文本内容似乎超出了容器宽度。
结合截图分析：
1.  长文本（如 URL）没有自动换行，撑开了容器宽度。
2.  `.detail-page` 宽度被内容撑开超过了视口宽度 (100vw)。
3.  但 `body` 的背景色是蓝色（`var(--qd-bg)` / `#007aff`）。
4.  当页面横向滚动或视口被撑大时，右侧露出了底层的蓝色背景。

## 解决方案
核心是修复 CSS 中的文本溢出处理。

1.  **强制文本换行**：在富文本容器 `.rich-content` 中添加 `word-break: break-all;` 或 `overflow-wrap: break-word;`，防止长链接/长单词撑开页面。
2.  **限制图片宽度**：确保富文本内的图片 `max-width: 100%;`（代码里已有，但需确认生效）。
3.  **防止容器溢出**：给 `.detail-page` 或 `.content-card` 添加 `overflow-x: hidden;` 作为兜底（虽然治标不治本，但能防意外）。
